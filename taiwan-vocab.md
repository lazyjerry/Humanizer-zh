# 繁體中文與台灣用語規範

所有輸出必須符合本規範。

---

## 核心原則

- 所有中文內容使用繁體中文，絕對禁止簡體字
- 強制使用台灣常用詞彙，避免中國大陸用語
- 技術專有名詞保留英文原文（API、Token、Commit 等）
- 避免使用 emoji

---

## 台灣用語對照表

| 情境 | 中國用語 | 台灣用語 |
|------|----------|----------|
| 軟體相關 | 軟件、程序、代碼、文檔、優化 | 軟體、程式、程式碼、文件、最佳化 |
| 網路相關 | 网络、服务器、在线、离线 | 網路、伺服器、線上、離線 |
| 檔案操作 | 文件夹、默认值、信息 | 資料夾、預設值、資訊 |
| 系統相關 | 操作系统、内存、硬盘、数据库 | 作業系統、記憶體、硬碟、資料庫 |
| 功能描述 | 打开、配置、视频、音频 | 開啟、設定、影片、音訊 |
| 介面相關 | 界面、窗口、菜单、鼠标 | 介面、視窗、選單、滑鼠 |
| 開發用語 | 进程、线程、移动端、调试 | 行程、執行緒、行動裝置、除錯 |
| 其他 | 激活、通过、用户、落地、智能 | 啟用、透過、使用者、實現、智慧 |

---

## 常見句式轉換

- 「通過 API 來實現」→「透過 API 實現」
- 「打开配置文件」→「開啟設定檔」
- 「默認情況下」→「預設情況下」
- 「查看文檔」→「查看文件」

---

## 禁用主觀形容詞

以下詞彙因過於主觀、誇大，應避免使用：專業的、企業級、強大的、優秀的、完善的、高效的、先進的、卓越的、最佳的、一流的、頂級的、極致的、完美的、絕佳的

---

## 格式規範

- 中文內容使用全形標點符號
- 英文內容保持半形標點符號
- 中英文之間適當留空格
- 數字表達：阿拉伯數字 + 中文單位（2025 年、3 個步驟）

---

## 保留原文的內容

以下內容維持英文原文，不進行翻譯：
- URL 和網址
- 檔案路徑
- 套件名稱
- Git 指令
- 環境變數名稱
- 技術專有名詞（API、CLI、Token、Commit、Pull Request、Webhook、Endpoint）
